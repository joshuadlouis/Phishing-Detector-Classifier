# Phishing Intent Classifier

## 🤔 What is this?
The **Phishing Intent Classifier** is a custom Natural Language Processing (NLP) pipeline designed to actively categorize malicious emails and texts based on the exact psychological manipulation tactic they employ. 

Instead of a standard binary spam filter ("Is this Safe or Phishing?"), this sequence attempts to understand the underlying behavioral vector of the attacker by mapping the communication directly to **Cialdini’s 6 Principles of Persuasion**.

The classifier evaluates raw text into 7 overlapping intentional categorizations:
1. `Urgency and Scarcity`
2. `Authority`
3. `Fear`
4. `Greed and Reciprocity`
5. `Commitment and Consistency`
6. `Consensus and Social Proof`
7. `Liking`

## 🎯 What is it for?
This repository was developed as an advanced academic NLP assignment. While most datasets offer simple binary mappings, actual Cybersecurity training relies upon understanding the psychological hooks standard employees fall for. 

By categorizing phishing attempts securely through offline models, we can develop advanced metrics to flag heavily targeted authoritative spear-phishing versus generalized urgency-based spam.

## 🧠 How is it being trained?
This pipeline utilizes an aggressive, multi-model transfer-learning workflow:

#### 1. Robust Data Ingestion
We pull directly from the massive `rokibulroni/Phishing-Email-Dataset` utilizing recursive `pathlib` mappings. Pandas is leveraged to compile structured CSVs alongside raw `.txt` files stripped directly of their native HTML tags using `BeautifulSoup`. Redundant text payloads are dropped.

#### 2. Auto-Labeling via Zero-Shot Classification
Because the original dataset only features simple "Phishing" vs. "Safe" mappings, we rely on **Hugging Face's Zero-Shot Classification Pipeline**.
We instantiate `facebook/bart-large-mnli` (a massive generalized language model) and feed it the raw text context along with our 7 custom Cialdini candidate labels. The model infers the strongest semantic alignment without previous specific intent-training and tags the text accordingly!

#### 3. Fine-Tuning a Fast Offline Artifact
Relying on a 1.6GB LLM for Zero-Shot text processing is slow and computationally expensive. To optimize, we take the outputs generated by the Zero-Shot model and use them as the "Ground Truth" foundation to train a much smaller, extremely fast architecture: `distilbert-base-uncased`.

Using Hugging Face's `Trainer` API, we run the newly labeled data through 3 specific epochs, altering the optimizer states until the DistilBERT neural weights effectively mimic the classification capabilities of the larger BART architecture. This results in the final `./intent_model/` folder—a compiled, production-ready artifact that classifies texts in merely ~20 milliseconds on a standard CPU!

---
*Note: Due to high computational costs, `build_classifier.py` limits the Zero-Shot processing loop to a random subset of 100 samples for pipeline validation purposes. To train the true, fully-weighted architecture, remove the `sample_limit` parameter in the execution script!*
